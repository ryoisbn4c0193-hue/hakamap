# ファイル・フォルダ選択設計

## 目的

既定ブラウザで動作するReact画面から、利用者が許可したローカルファイルまたはフォルダを
安全に選択し、絶対パスをブラウザへ公開せずJavaバックエンドで処理する。

## MVPの実装方式

- Java標準のSwing `JFileChooser`を使用する。
- Swingのイベントディスパッチスレッド（EDT）上でダイアログを生成し、表示する。
- ファイル選択とフォルダ選択の両方に対応する。
- JavaプロセスをGUIダイアログを表示可能な状態で起動し、ヘッドレスモードでは実行しない。
- ダイアログがブラウザの背面へ隠れないよう、表示時の前面化とフォーカス移動を考慮する。
- Swingの外観はWindows標準ダイアログと完全には一致しないことを許容する。

## Reactとの境界

- Reactは用途を指定して選択開始APIを呼び出す。
- 同時に複数の選択ダイアログを開かず、選択中は同じ操作を二重実行できないようにする。
- 選択された絶対パスはJavaバックエンドの一時的な選択コンテキストだけに保持する。
- Reactには絶対パスを返さず、選択結果を識別する一時的なIDと表示に必要な最小限の情報だけを返す。
- 後続の取り込み、プロジェクト作成、再関連付けなどは一時選択IDを指定して実行する。
- 一時選択IDはJavaバックエンドが推測困難なランダムUUIDとして生成する。
- 一時選択IDを発行したHTTPセッションおよび背景、添付、保存先などの選択用途へ紐づけ、
  別のセッションまたは用途では使用できないようにする。
- 一時選択IDの有効期限は発行から5分とする。
- 後続処理で正常に使用した時点で無効化し、同じIDを再利用できない1回限りの値とする。
- 選択キャンセル、プロジェクト終了、およびアプリケーション終了時にも、関連するIDを無効化する。
- 有効期限切れまたは無効なIDを指定した場合は処理を開始せず、利用者へ再選択を求める。
- 一時選択IDを使用する時点で、対象の存在、正規化したパス、ファイル種別、拡張子、
  MIMEタイプ、サイズ、および用途別の許可条件を再検証する。
- 一時選択IDと選択した絶対パスをログへ記録しない。
- 利用者によるキャンセルは業務エラーにせず、「選択なし」として返す。
- ダイアログ表示またはファイルアクセスに失敗した場合は、個人情報や絶対パスを含めない
  Problem Detailsを返す。

## 差し替え可能性

- ファイル選択処理をバックエンド内部のインターフェースとして分離する。
- Windows標準と完全に同じ外観や追加機能が必要になった場合は、将来JNA経由の
  Windows `IFileDialog`実装へ差し替えられるようにする。
- MVPではJavaFXとJNAをファイル選択のためだけに追加しない。

## 検証要件

- ファイルおよびフォルダを選択し、一時選択IDを使って後続処理を実行できることを確認する。
- キャンセルがエラーにならず、対象操作が実行されないことを確認する。
- 複数のダイアログが同時に開かず、選択操作を二重実行できないことを確認する。
- APIレスポンス、Problem Details、およびログへ絶対パスが含まれないことを確認する。
- Windows上でダイアログが視認可能な前面位置に表示され、キーボードでも操作できることを確認する。
- 別セッション、別用途、有効期限切れ、および使用済みの一時選択IDが拒否されることを確認する。
- IDの利用時に対象が再検証され、選択後に差し替えられた不正なファイルが拒否されることを確認する。
- キャンセル、プロジェクト終了、およびアプリケーション終了で関連IDが無効化されることを確認する。
