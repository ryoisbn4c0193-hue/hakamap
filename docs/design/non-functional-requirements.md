# 非機能要件

## 利用形態

- 基本的に同一人物が利用し、複数利用者による同時更新を保証しない。
- アプリケーション独自のログイン機構およびユーザー管理機能を設けない。
- 作成者および更新者などのユーザー情報を保持しない。
- データの監査用メタデータとして保持するのは、作成日時`created_at`と更新日時
  `updated_at`だけとする。
- プロジェクトと人物には`created_at`と`updated_at`の両方を保持する。

## 想定データ量

- 1プロジェクトあたり最大5,000墓所での利用を想定する。
- 5,000墓所は性能設計上の想定最大件数であり、入力を物理的に禁止する上限ではない。
- 人物は5,000墓所に平均4人、合計20,000人を性能試験上の想定最大とする。
- 20,000人は性能保証の基準であり、1墓所またはプロジェクト全体の人物登録を禁止する
  固定上限として扱わない。

## 性能目標

- 5,000墓所を含むプロジェクトで、次の通常操作を1秒以内に応答することを目標とする。
  - 墓所およびエリアの選択と移動
  - サイズ変更
  - ズーム、パン、およびスクロール
  - UndoおよびRedo
  - 検索
  - プロパティパネルおよびラベル表示の切り替え
- 墓所、エリア、人物、添付メタデータ、および設定を含む5,000墓所のプロジェクトを、
  5秒以内で保存できることを目標とする。
- 大容量添付ファイル本体の新規コピーなど、ストレージ性能へ大きく依存する処理は
  通常の保存時間目標に含めない。
- 背景、エリア、墓所、人物、および添付メタデータを含む5,000墓所のプロジェクトを、
  5秒以内で読み込み、編集を開始できることを目標とする。
- 添付ファイル本体を読み込むタイミングは設計で決定する。
- 20,000人を含む5,000墓所を対象とする検索結果を1秒以内で表示することを目標とする。
- アプリケーション起動から編集可能になるまで5秒以内を目標とし、デフォルト
  プロジェクトがある場合は、その読み込みと編集画面表示を含める。
- 通常利用時のメモリ使用量は、5,000墓所、背景画像1枚、人物情報、および添付
  メタデータを含む状態で1 GB以下を目標とする。
- インポートおよびエクスポートなど、一時的にメモリ使用量が増える処理は通常利用時の
  メモリ目標に含めない。

## 動作環境

- MVPの動作保証対象はWindows 11だけとする。
- Windows 10、macOS、およびLinuxはMVPのサポート対象外とする。
- ブラウザはEdge、Chrome、およびFirefoxのリリース時点の最新安定版と1世代前を対象とし、
  Windows 11標準のEdgeを基準とする。

| 項目 | 最低動作環境 | 推奨動作環境 |
| --- | --- | --- |
| OS | Windows 11 | Windows 11 |
| CPU | Intel Core i3 第10世代相当 | Intel Core i5 第12世代相当以上 |
| メモリ | 8 GB | 16 GB |
| ストレージ空き容量 | 2 GB以上 | 10 GB以上 |
| ディスプレイ解像度 | 1366×768以上 | 1920×1080以上 |

- 1366×768でも主要操作を行えるようにする。
- 1920×1080以上を快適に利用できる推奨環境とする。
- 低解像度ではパネルの折りたたみなどによって操作領域を確保する。

## 自動復旧

- 異常終了に備え、未保存変更がある間は30秒ごとに編集中データを一時保存する。
- 異常終了後の次回起動時に復旧データを検出する。
- 復旧データがある場合は利用者へ復元するか確認し、利用者が破棄することもできるようにする。
- 自動復旧データは、通常保存データおよび自動バックアップと分けて管理する。
- 復旧JSONはプロジェクトごとに最新1世代を保持し、手動保存または変更破棄の成功後に削除する。
- 復旧データの検証に失敗した場合は適用せず、正式保存データを変更しない。

## ログ

- 障害解析および保守を目的としてアプリケーションログを出力する。
- 起動、終了、プロジェクトの作成・読み込み・保存、バックアップ、復元、インポート、
  エクスポート、自動復旧、エラー、および例外を記録する。
- 墓所選択、ズーム、パン、墓所の細かな移動など、頻繁な通常編集操作を逐一記録する
  監査ログは設けない。
- ログは期間固定ではなく、ファイルサイズと世代数によって管理する。
- ログはUTF-8のテキストとし、1ファイル最大10MB、現行1ファイルと過去5世代、
  最大容量の目安を60MBとする。
- 上限を超える場合は古いログから削除する。
- 個人情報および利用者が入力した非公開情報を不要に記録しない。
- 絶対パス、一時選択ID、HTTPセッション情報、および通常の編集履歴を記録しない。
- ログ書き込み失敗だけを理由に編集データまたは保存済みデータを失わない。

## エラー処理

- エラー発生時は、ダイアログなどによって利用者へ内容を通知する。
- エラー内容は、個人情報を含めない形でアプリケーションログへ記録する。
- 復旧可能なエラーでは該当処理だけを中断し、アプリケーションを継続動作させる。
- 復旧不能な致命的エラーの場合だけアプリケーションを終了する。
- 業務エラーとシステムエラーを区別できるようにする。
- 利用者向けメッセージへ内部例外やスタックトレースをそのまま表示しない。
- 詳細なエラー分類とメッセージ一覧はエラー設計で定める。

## セキュリティ

- プロジェクトデータおよびバックアップへアプリケーション独自の暗号化を適用しない。
- 保存データへのアクセス制御はWindowsのユーザー認証とファイルアクセス制御を
  主な保護境界とする。
- 暗号化機能はMVPの対象外とする。
- 独自のファイル形式を採用する場合も、それを暗号化とはみなさない。
