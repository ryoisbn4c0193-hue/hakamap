# フロントエンド状態管理詳細設計

## 目的

Javaバックエンドの確定状態、Reactの表示状態、入力途中のフォーム、およびPixiJSの
操作プレビューを分離し、非同期応答、入力破棄、リビジョン競合、および操作失敗が
発生しても、利用者の意図しない上書きや入力消失を防ぐ。

## 状態の正と責務

| 状態 | 管理先 | 保存・revision対象 |
| --- | --- | --- |
| Project、背景、エリア、墓所、人物、添付、導出状態 | Javaバックエンド | 対象 |
| APIから取得した確定スナップショットと差分 | TanStack Query | Java状態のクライアント側写像 |
| 選択、ズーム、パン、パネル、タブ、検索条件 | Zustand | 対象外 |
| フォームの入力途中の値と検証表示 | React Hook Formまたはコンポーネント状態 | 確定まで対象外 |
| ドラッグ、サイズ変更、矩形選択の途中候補 | PixiJSアダプターと操作セッション | 確定まで対象外 |
| Operationの進捗 | TanStack Query | Operation結果の確定時だけ対象 |

- 保存対象データの正はJavaだけに置き、ZustandへProject全体を複製しない。
- TanStack Queryのキャッシュを永続化せず、ブラウザ再表示時はスナップショットを再取得する。
- PixiJSへドメインオブジェクトを直接保持させず、描画用の読み取り専用モデルへ変換する。
- 業務ルールをフロントエンドだけで確定せず、操作確定時にJavaで再検証する。

## TanStack Queryの構成

### Queryキー

```text
['projectSnapshot', projectId]
['projectHistory', projectId]
['projectSearch', projectId, searchSessionId]
['operation', operationId]
['catalogProjects']
```

- Project UUIDを含まない曖昧なキャッシュキーを使用しない。
- Projectを閉じる、切り替える、またはアプリを終了するときは、そのProjectのQueryを
  キャンセルしてキャッシュから削除する。
- 検索カーソルと結果一覧は編集スナップショットから分離し、revision競合時は破棄して
  最初のページから検索し直す。

### スナップショットと差分

- Projectを開いた直後は`ProjectSnapshotResponse`を取得し、取得完了までは編集操作を
  有効にしない。
- コマンド、Undo、およびRedo成功時は、応答のupsert、削除UUID、導出状態、
  `revision`、`dirty`、および履歴概要を1回のキャッシュ更新で反映する。
- 差分適用中にReactがProjectの一部だけを観測しないよう、Queryキャッシュ更新関数内で
  新しいスナップショット写像を組み立てて置換する。
- 差分の対象UUIDが現在キャッシュと整合しない場合は推測で補完せず、編集操作を止めて
  スナップショットを再取得する。
- API成功後の確定値を使用し、送信前の文字列、丸め前の座標、およびクライアント生成候補を
  確定状態として残さない。

## ZustandのUI状態

```text
selectedEntityIds
primarySelectionId?
activeSidebarFilter
activePropertyTab
leftPanelCollapsed
rightPanelCollapsed
viewport
labelDisplayMode
activeTool
interactionMode
openDialog?
notificationQueue
```

- `selectedEntityIds`はUUIDだけを保持し、エンティティ本体はQueryキャッシュから参照する。
- 単一選択では`primarySelectionId`を設定し、複数選択では最後に主選択となったUUIDだけを
  補助的に保持する。
- 差分適用で選択中エンティティが削除された場合は、そのUUIDを選択から除外する。
- Project切り替え時は選択、検索、ダイアログ、操作候補、およびviewportを初期化し、
  別Projectへ持ち越さない。
- Zustandの開発者向けミドルウェアを本番ログや永続ストレージへ接続しない。

## フォーム入力

- 墓所基本情報、エリア情報、人物、および添付表示情報は、ダイアログまたはパネルを
  開いた時点のサーバー確定値からフォーム初期値を作る。
- 入力中はProjectのQueryキャッシュを変更せず、「保存」または「適用」でコマンドを送る。
- フォームのdirtyは入力開始時のフォーム値との比較で判定し、Project全体の`dirty`と区別する。
- クライアント側検証は早期案内に使用するが、Javaの検証結果を置き換えない。
- API成功時はサーバー確定値でフォームを再初期化し、フォームdirtyを解除する。
- API業務エラー時は入力を保持し、Problem Detailsの安全なフィールド違反を対応項目へ表示する。

### 選択変更と入力破棄

- フォームに未確定入力がない場合は、地図またはサイドバーから直ちに選択を変更できる。
- 未確定入力がある状態で別エンティティの選択、選択解除、パネル閉鎖、Project切り替え、
  またはアプリ終了を開始した場合は、入力破棄の確認を表示する。
- 確認は「入力内容を破棄して移動」と「キャンセル」を基本とする。
- キャンセルでは選択、フォーム、viewport、およびサーバー状態を変更しない。
- 人物・添付ダイアログを閉じる場合も同じ判定を使用し、ブラウザ標準確認とアプリ内確認を
  二重表示しない。

## 地図操作プレビュー

- ドラッグ、サイズ変更、および背景変形の開始時に、対象UUID、開始形状、
  開始`revision`、ポインター情報を持つ一時`InteractionSession`を作る。
- 操作中はPixiJSの描画候補だけを変更し、QueryキャッシュとJava状態を変更しない。
- ポインター確定時は最終候補からコマンドpayloadを作り、同種の競合操作を無効化する。
- 成功時はAPI差分を反映してプレビューを破棄する。
- エラー、キャンセル、フォーカス喪失、または送信前に対象が消えた場合はプレビューを破棄し、
  Queryキャッシュの確定値へ表示を戻す。
- 操作量0などJavaが`noChange`を返した場合も確定値へ戻し、履歴を増やさない。

## API処理中の操作制御

- 同じエンティティまたは同じProject集約を変更するコマンドの多重送信を防ぐ。
- コマンド送信中もズームとパンは許可するが、対象の再編集、Undo、Redo、保存、
  Project切り替え、および終了は応答完了まで無効化する。
- 全画面を一律にブロックせず、無効な操作には処理中であることを視認可能に示す。
- 長時間Operationでは進捗とキャンセル可否を表示し、`committing`以降は
  キャンセル操作を無効化する。
- 通信エラーで結果が不明な状態では同じコマンドを自動再送しない。スナップショットを
  再取得し、結果を確認できるまで変更操作を停止する。

## リビジョン競合

- `project-revision-conflict`を受けた場合は、コマンドを自動再実行しない。
- 地図操作のプレビューは破棄して最新スナップショットを取得する。
- フォーム操作では送信時の入力値を一時退避し、最新スナップショット取得後も保持する。
- 最新値と退避値を確認できる画面を表示し、利用者が次を選択できるようにする。
  - 最新値を採用して退避値を破棄する。
  - 退避値をフォームへ戻し、最新`revision`で改めて適用する。
  - 再適用せず、退避値を選択・コピーできる状態で残す。
- 再適用でもJava側の全検証と必要な配置警告確認をやり直す。
- 削除、一括操作、移動、サイズ変更、コピー、採番、および背景操作は入力値の
  自動再適用対象にしない。

## 通知とエラー表示

- 成功通知は操作結果を簡潔に示し、個人名、管理番号、ファイル名などを通知履歴へ複製しない。
- フォーム項目エラー、操作全体の業務エラー、保存・接続障害、致命的障害を表示上で区別する。
- 同じ原因の通知を連続追加せず、処理中表示を完了または失敗表示へ更新する。
- Problem Detailsの未知`code`は安全な共通エラーとして扱い、`detail`をHTMLとして描画しない。

## コンポーネント境界

```text
AppShell
├─ ProjectLifecycleCoordinator
├─ EditorLayout
│  ├─ Sidebar
│  ├─ MapViewport
│  │  └─ PixiMapAdapter
│  └─ PropertyPanel
│     ├─ GraveBasicForm
│     ├─ PeopleTab
│     ├─ AttachmentsTab
│     └─ HistoryTab
├─ DialogHost
└─ NotificationHost
```

- `ProjectLifecycleCoordinator`はProjectのopen、close、切り替え、未保存確認を調整する。
- `MapViewport`はReactのレイアウトとアクセシビリティ境界を担当し、PixiJS APIを
  他のコンポーネントへ公開しない。
- フォームコンポーネントはDTO組み立てを直接行わず、機能単位のmutation hookを呼び出す。
- APIクライアント、DTOのZod検証、差分適用、およびProblem Details変換を
  コンポーネントから分離する。

## 検証要件

- API差分適用後のQueryキャッシュがスナップショット再取得結果と一致することを確認する。
- ZustandへProjectエンティティや個人情報を重複保持しないことを確認する。
- 未確定フォームがある選択変更で破棄確認が表示され、キャンセル時に状態が変わらないことを確認する。
- 地図プレビュー中にQueryキャッシュとrevisionが変化しないことを確認する。
- コマンド成功、no-op、業務エラー、通信エラー、および競合でプレビューが確定状態へ収束することを確認する。
- 競合時にフォーム入力を保持でき、地図操作と一括操作が自動再実行されないことを確認する。
- Project切り替え後に前Projectの選択、検索結果、フォーム、ダイアログ、および操作候補が残らないことを確認する。
- コマンド多重送信、Undoとの競合、および結果不明時の自動再送が起きないことを確認する。
- キーボードだけで破棄確認、エラー確認、およびフォーム再適用を操作できることを確認する。
