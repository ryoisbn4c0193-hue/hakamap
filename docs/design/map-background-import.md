# 地図背景インポート設計

## 目的

利用者が所有または利用権限を持つ航空写真や墓地図面を、外部サービスへ送信せずに
墓地マップの背景として取り込めるようにする。

## 対応形式

初期版では次のローカルファイルを受け付ける。

- PNG
- JPEG
- WebP
- PDF

PDFは文書やベクターデータとして編集せず、1ページだけのPDFをPNGへ変換し、
通常の背景画像として扱う。

## 共通の取り込み処理

- 利用者がファイル選択画面から指定したローカルファイルだけを取り込む。
- URLの指定やオンラインサービスからのダウンロードには対応しない。
- 拡張子だけで形式を判断せず、実際のファイル内容、寸法、容量を検証する。
- 対応形式と一致しないファイル、破損したファイル、または安全に処理できない
  ファイルを拒否する。
- 取り込みおよび変換はアプリケーション管理領域の編集セッション別一時領域で完了させ、
  検証後も手動保存まではステージング状態で保持する。正式な`assets`への反映は手動保存時に行う。
- 途中で失敗または中断した場合は、未完成のアセットを地図背景として登録しない。

## PDFの取り込み

### 処理手順

1. PDFの基本的な妥当性とページ数を検証する。
2. 複数ページである場合は業務エラーとして取り込みを中止する。
3. 1ページ目を安全にプレビューする。
4. そのページを制限された解像度でPNGへ変換する。
5. 生成したPNGをステージングし、手動保存時にプロジェクトの`assets/backgrounds`へ反映する。
6. 生成したPNGを背景画像として登録する。

### 制限

- 背景として取り込めるPDFは1ページだけのPDFとする。
- 複数ページPDFは業務エラーとして拒否する。
- パスワード保護または暗号化されたPDFは初期版では対象外とする。
- PDF内のJavaScript、リンク、フォーム、動画、音声、添付ファイルを実行または展開しない。
- PDF内の文字、線、図形、画像を編集可能な地図要素として抽出しない。
- ベクターPDFも選択ページ全体をラスタライズし、1枚のPNGとして扱う。
- 変換元のPDFはプロジェクトへ永続保存せず、変換後のPNGだけを正式アセットとして保存する。
- アセットメタデータには元ファイル名と取り込み元がPDFであることを保持する。
- 外部PDFビューアー、プリンター、オンライン変換サービスへ処理を委譲しない。

### 変換処理

- PDFからPNGへの変換はアプリケーション内で行う。
- PDF変換ライブラリは、Java 21との互換性、Windows配布物への同梱可否、
  オフライン動作、ライセンス、および安全な解析機能を確認して選定する。
- 出力画像の辺の長さ、総画素数、および推定メモリ使用量に上限を設ける。
- 背景の入力ファイルは1ファイル最大100MBとする。
- 背景画像およびPDFから生成する表示用画像は最大1億画素とし、幅および高さを
  それぞれ最大30,000pxとする。
- 上限を超えるファイルは自動縮小せず、理由を表示して取り込みを拒否する。
- 変換時に外部フォントや外部リソースをダウンロードしない。

## 背景画像の配置

- 背景画像は左上を原点、右をX正方向、下をY正方向とする地図座標系へ配置する。
- 地図座標の1単位は1ピクセル相当の論理単位とし、画面ピクセルおよびズーム率とは分離する。
- 利用者は背景画像の位置、回転、およびX・Y方向の表示倍率を調整できる。
- 背景の位置は`x`と`y`、時計回りの回転角度は`rotation`、X・Y方向の倍率は
  `scaleX`と`scaleY`として保存する。倍率の初期値は1とする。
- 元画像のピクセル座標から地図座標への変換情報を保存する。
- 区画、通路、マーカーなどの編集要素は背景画像とは独立した地図座標として保存する。
- 背景画像の移動、再調整、交換によって、既存の編集要素を自動的に移動しない。
- 既存要素も一緒に変換する機能が必要になった場合は、明示的な別操作として設計する。

## 設計候補（未確定）

次の内容は既存構想として保持するが、確定要件ではない。採用可否は基本設計で決定する。

- PNG、JPEG、およびWebPの取り込み元と別に表示用画像を生成し、両方を永続保存する方式
- 背景画像の透明度調整
- 画像上の2点と実距離による縮尺設定
- 上限超過時に、安全な解像度への縮小候補を提示する機能

## プライバシーと権利

- 取り込んだファイルと生成した画像を端末外へ送信しない。
- 取り込んだファイル名や内容をログへ記録しない。
- 利用者が取り込む画像や図面について、所有権または必要な利用許諾を確認する責任が
  利用者にあることを画面上で説明する。
- Google Mapsなど、保存や派生データ作成が制限されるサービスの画面を取り込むための
  専用機能を提供しない。

## 検証要件

- 対応するPNG、JPEG、WebPをオフラインで取り込めることを確認する。
- 1ページだけのPDFをPNGへ変換できることを確認する。
- 複数ページPDFが業務エラーとして拒否されることを確認する。
- PDF内の能動的コンテンツが実行されないことを確認する。
- 暗号化PDF、破損ファイル、偽装された拡張子、上限を超える画像が拒否されることを
  確認する。
- 100MB、1億画素、または各辺30,000pxのいずれかを超える背景が拒否されることを確認する。
- 取り込み中の強制終了を模擬し、未完成のアセットが登録されないことを確認する。
- 背景画像を変更しても、既存の区画やマーカーの保存座標が変化しないことを確認する。
- 背景の位置、時計回りの回転、および正数のX・Y倍率が保存後に同じ配置として復元されることを確認する。

## 採用しない方式

- PDFを編集可能な文書またはベクター地図として扱う方式
- 複数ページPDFからページを選択して取り込む方式
- PDFの全ページを1つの背景として自動的に結合する方式
- PDF内のスクリプト、リンク、フォーム、添付ファイルを利用する方式
- オンライン変換サービスまたは外部アプリケーションへの依存
- 取り込み元の絶対パスを永続化し、元ファイルを直接参照し続ける方式
