# オフライン動作設計

## 目的

Hakamapのすべての機能を外部サービスへ依存せずに提供し、墓地に関する非公開情報が
利用者の意図なく端末外へ送信されないことを保証する。

## 決定事項

### 実行時のネットワーク利用

- パッケージ版の全機能を、インターネット接続がない状態で利用可能にする。
- Google Mapsを含む外部地図APIを使用しない。
- 更新確認、ライセンス確認、テレメトリー、広告配信、クラッシュレポート送信を行わない。
- 実行時に外部の認証サービス、ストレージ、データベースへ接続しない。
- アプリケーションが行うHTTP通信は、ブラウザと`127.0.0.1`上のHakamap
  バックエンド間の通信に限定する。
- MVPではアプリケーション内および添付PDFから外部Webリンクを開く機能を設けない。
- 自動アップデートの有無と更新版の適用方法は、配布方式とともに基本設計で決定する。

### 実行時アセット

- JavaScript、CSS、フォント、アイコン、画像、その他のUIアセットを配布物へ同梱する。
- CDN、外部フォント、外部スクリプト、外部地図タイルを使用しない。
- Content Security Policyによって、通信先を原則として同一オリジンに限定する。
- 外部URLを墓地マップの背景またはプロジェクトアセットとして登録しない。

### 地図背景の取り込み

- 墓地マップの背景には、利用者が所有または利用権限を持つ航空写真、図面、画像を
  ローカルファイルから取り込めるようにする。
- 取り込んだファイルはHakamapの管理対象領域へコピーし、元ファイルの場所や存在に
  依存しないようにする。
- 取り込み時にネットワーク上のURLからファイルをダウンロードしない。
- 利用者が選択したファイルの権利、利用条件、および第三者への提供可否は利用者が
  確認するものとする。
- Google Mapsなど、保存や再利用が制限される外部サービスの表示内容を取り込むための
  専用機能を設けない。
- 対応形式、画像変換、寸法、座標合わせの方法は、地図背景インポート設計で定める。

### ビルド時との分離

- Maven、pnpm、JavaScriptパッケージ、Windowsパッケージ作成ツールなどの取得は
  開発時およびビルド時に限り許可する。
- ビルド時に取得した依存関係を配布物へ必要な範囲で同梱し、実行時に追加取得しない。
- 新しい依存関係を追加する際は、オフライン実行への影響と再配布ライセンスを確認する。

### 利用者のブラウザとの境界

- Hakamapは利用者の既定ブラウザをUI表示に使用する。
- ブラウザ自身の同期、更新確認、拡張機能、その他の通信はHakamapの制御対象外とする。
- Hakamapの画面から外部コンテンツを読み込まず、墓地に関する情報をブラウザ経由で
  外部へ送信しない。

## 検証要件

- ネットワーク接続を無効にしたWindows環境で、インストール、起動、地図編集、保存、
  再起動、検索、バックアップ、復元を実行できることを確認する。
- パッケージ版の通常操作で、ループバック以外への接続が発生しないことを確認する。
- 配布物を新しいWindows環境へインストールした際、実行時に追加の依存関係を
  ダウンロードしないことを確認する。
- 外部URLを含む画像参照など、不正なプロジェクトデータを読み込んでも外部通信を
  開始しないことを確認する。
- Content Security Policyによって、意図しない外部スクリプト、画像、接続が拒否される
  ことを確認する。

## 採用しない方式

- Google Mapsなどの外部地図APIへの実行時接続
- 外部地図タイルの表示、キャッシュ、またはオフライン保存
- CDNからの実行時アセット取得
- 外部URLを参照し続ける地図背景
- 利用者の操作を記録または送信するテレメトリー
