# フロントエンドUI設計

## 目的

Hakamapの画面全体で一貫した操作性とアクセシビリティを提供し、個別画面ごとの
重複したUI実装を抑える。

## 採用ライブラリ

- React向け標準UIコンポーネントとしてMUIのMaterial UIを使用する。
- パッケージは`@mui/material`を使用する。
- Material UIの標準スタイルエンジンであるEmotionを使用し、`@emotion/react`と
  `@emotion/styled`を依存関係に含める。
- アイコンが必要な場合は、再利用可能な同梱SVGまたは`@mui/icons-material`を使用する。
- MUI Xの有償機能を初期版の前提にしない。
- 依存関係のバージョンはpnpmのロックファイルで固定する。

Material UIおよび標準の依存関係はビルド時に取得し、Reactの本番成果物へ含める。
実行時にMUIのCDNへ接続しない。

## コンポーネント利用方針

- ボタン、フォーム、ダイアログ、メニュー、タブ、通知、一覧などは、独自実装より先に
  Material UIの既存コンポーネントを検討する。
- アプリケーション固有の複合操作は、Material UIコンポーネントを組み合わせた共通
  コンポーネントとして実装する。
- MUIコンポーネントを直接利用する画面ごとに、同じ見た目のスタイル指定を重複させない。
- PixiJSで描画する地図キャンバス自体はMUIへ置き換えない。
- 地図ツールバー、プロパティパネル、ダイアログ、フォームなど、キャンバス周辺のUIには
  Material UIを使用する。
- MUIの内部DOM構造や自動生成クラス名に依存した実装を避け、公開されたプロパティ、
  `sx`、`styled`、テーマのコンポーネント設定を使用する。

## テーマ

- アプリケーションルートに`ThemeProvider`と`CssBaseline`を配置する。
- 色、タイポグラフィ、余白、角丸、影、フォーカス表示などの共通値をテーマへ集約する。
- UI文言と日本語表示に適した、Windowsへ標準搭載されるシステムフォントを優先する。
- Google Fontsなどの外部フォントを実行時に取得しない。
- フォントを追加する場合は、再配布ライセンスを確認して配布物へ同梱する。
- 初期版はライトテーマを基本とする。ダークテーマは別途要件が決まるまで必須としない。

## 編集競合ダイアログ

- リビジョン競合時は入力中の値を一時保持し、最新状態を取得して競合を通知する。
- 最新内容を採用して入力を破棄する、最新内容を確認して入力を再適用する、または
  再適用せず入力値をコピー可能な形で退避する、のいずれかを選択できるようにする。
- 再適用時は最新リビジョンを使ってバックエンドで再検証する。
- 地図操作、削除、および一括操作は自動再実行しない。
- ダイアログを閉じるだけで入力内容を失わないよう、明示的な選択を要求する。

## 履歴タブ

- 最大100件の操作履歴を新しい順に表示する。
- 操作日時、個人情報を含まない操作説明、Undo済みまたは適用中の状態、および
  「ここまで保存済み」の位置を表示する。
- Undo／Redoは順番どおりに実行し、任意時点へのジャンプや履歴の手動削除は設けない。

## 人物一覧

- 1墓所に多数の人物が登録されても描画要素数が過大にならないよう、一覧は仮想スクロールを使用する。
- 現在選択した墓所の表示に必要な人物だけを取得し、プロジェクト内の人物全件をReactへ常時保持しない。
- 人物は`displayOrder`による登録順で表示し、新規人物を末尾へ追加する。
- 編集では順序を変えず、削除時は残りを詰め、Undo時は元の位置へ戻す。
- MVPでは人物の手動並べ替え操作を設けない。

## 添付一覧

- 添付は`displayOrder`順に表示し、新規追加時は末尾へ配置する。
- ポインターではドラッグ＆ドロップ、キーボードでは「上へ」「下へ」で並べ替えられるようにする。
- 並べ替えをUndo／Redo対象の1操作としてバックエンドへ送信する。
- 削除後は残りの順序を詰め、Undo時は元位置へ戻す。

## アクセシビリティ

- Material UIを採用するだけでアクセシビリティ対応が完了したとはみなさない。
- すべての操作に適切な日本語ラベルを付ける。
- キーボードだけで主要操作へ到達し、実行できるようにする。
- 視認可能なフォーカス表示を維持し、テーマ変更で消さない。
- アイコンだけのボタンにはアクセシブルな名前とツールチップを付ける。
- ダイアログのフォーカス管理、入力エラーの関連付け、色のコントラストをテストする。
- 色だけで選択状態、エラー、カテゴリを伝えない。
- 地図キャンバスで選択した要素を、キーボードで操作可能なReact UIからも確認・編集
  できるようにする。

## オフライン動作

- Material UI、Emotion、アイコン、フォントをCDNから読み込まない。
- MUIの利用によって外部サービスへの実行時通信を追加しない。
- 必要なJavaScriptとCSSはViteでReactの本番成果物へまとめ、Spring Bootから配信する。
- 外部画像URLや外部フォントURLをテーマへ設定しない。

## 対応ブラウザ

- Windows 11標準のMicrosoft Edgeを基準ブラウザとする。
- Microsoft Edge、Google Chrome、およびMozilla Firefoxについて、アプリケーションの
  リリース時点における最新安定版と1つ前の安定メジャーバージョンを動作保証対象とする。
- SafariおよびInternet ExplorerはMVPの動作保証対象外とする。
- 対応対象外のブラウザが既定の場合もまず画面を開き、必要なWeb機能を確認する。
- 必須機能が不足する場合は編集画面を開始せず、Edge、Chrome、またはFirefoxの対応版で
  開くよう利用者へ案内する。
- ファイルおよびフォルダ操作はJavaバックエンドで行い、特定ブラウザだけのファイルシステムAPIへ依存しない。

## テスト

- React Testing Libraryでは、MUIの内部構造やCSSクラスではなく、ロール、ラベル、
  表示文言など利用者が認識する情報を基準に操作する。
- テーマを適用した状態でコンポーネントテストを実行する。
- キーボード操作、フォーカス移動、ダイアログ、フォームバリデーションをテストする。
- 重要な画面はWindows上のEdge、Chrome、およびFirefoxの対象バージョンで表示と操作を確認する。

## ライセンス

- Material UIのMITライセンス表示を配布物の第三者ライセンス一覧へ含める。
- MUI Xなど、Material UI本体と異なるライセンス条件を持つパッケージを追加する場合は、
  導入前に利用条件と配布条件を確認する。
