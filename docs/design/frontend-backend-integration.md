# フロントエンド・バックエンド統合設計

## 目的

開発時の操作性を維持しながら、パッケージ版ではReactの画面とSpring BootのAPIを
単一プロセスかつ同一オリジンで提供する。

## 決定事項

### 開発時の構成

- ReactはVite開発サーバー、APIはSpring Bootとして個別に起動する。
- Viteは`127.0.0.1:5173`、Spring Bootは`127.0.0.1:8080`を既定値とする。
- Viteは`/api`へのリクエストをSpring Bootへプロキシする。
- フロントエンドのアプリケーションコードには、開発用バックエンドのホスト名や
  ポート番号を埋め込まない。
- ViteとSpring Bootは、開発時もループバックアドレスだけにバインドする。

### 本番ビルド

- Maven Wrapperによる本番ビルドの一部として、固定されたpnpmを使用してReactを
  ビルドする。
- フロントエンドのテスト、型検査、またはビルドが失敗した場合は、Mavenの本番
  ビルド全体を失敗させる。
- Reactの生成物は`frontend/dist`や`backend/src/main/resources`へ永続的にコピーせず、
  Mavenのビルドディレクトリ配下で処理する。
- Reactの生成物をSpring Bootの実行可能JARへ静的リソースとして同梱する。
- 生成した静的リソースやJARをGitへコミットしない。
- 使用するNode.jsおよびpnpmのバージョンをリポジトリ内の設定で固定する。

### パッケージ版の実行時構成

- パッケージ版ではVite、Node.js、pnpmを起動または同梱しない。
- Spring BootがReactの静的ファイルと`/api`を同じHTTPサーバーから配信する。
- UIとAPIを同一オリジンとし、CORSを必要としない構成にする。
- APIエンドポイント、アセット、その他の予約済みパスを除き、クライアント側の
  画面URLへのGETリクエストはReactの`index.html`へフォールバックする。
- フォールバック処理によって、存在しないAPIや静的アセットのエラーが
  `index.html`へ置き換わらないようにする。
- 実行時は外部CDNや外部Webサーバーからファイルを取得しない。

## ビルド環境と配布物の境界

- Node.jsとpnpmは開発環境およびパッケージ作成環境だけで必要とする。
- 利用者のWindows環境にはNode.js、pnpm、Viteを要求しない。
- 利用者のWindows環境にはJavaの事前インストールを要求せず、後続のWindows
  パッケージ設計で最小化したJavaランタイムを同梱する。
- Mavenからフロントエンドを実行する仕組みは、実装時に対応バージョン、ライセンス、
  Windows上での再現性を確認して選定する。

## 検証要件

- 開発時にVite経由の`/api`アクセスが動作することを確認する。
- Maven Wrapperだけでフロントエンドを含む本番JARを生成できることを確認する。
- 本番JARの起動後、React画面、静的アセット、`/api`を同じオリジンから利用できる
  ことを確認する。
- 画面を直接再読み込みした場合に、Reactのクライアント側ルートが表示されることを
  確認する。
- 存在しないAPIが適切なHTTPエラーを返し、React画面へフォールバックしないことを
  確認する。
- インターネット接続を無効にした状態でも、本番JARの全画面と同梱アセットを利用
  できることを確認する。

## 採用しない方式

- パッケージ版でVite開発サーバーを起動する方式
- 利用者にNode.jsまたはpnpmのインストールを要求する方式
- Reactの生成物をソースディレクトリへコピーしてGit管理する方式
- UIとAPIを異なるオリジンで配信し、制限のないCORSで接続する方式
