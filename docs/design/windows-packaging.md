# Windowsパッケージ設計

## 目的

HakamapをJavaやNode.jsの事前導入なしでWindows 11へインストールし、アプリケーションの
更新および削除から利用者のプロジェクトデータを分離する。

## 配布形式

- Windows上で`jpackage`を使用し、`.exe`インストーラーを作成する。
- ユーザー単位でインストールし、原則として管理者権限を要求しない。
- `jlink`でHakamapの実行に必要なモジュールだけを含むJava 21ランタイムを作成し、同梱する。
- 利用者の端末にJava、Node.js、pnpm、Vite、およびMavenの事前インストールを要求しない。
- Windows向け成果物はWindows上で作成し、Linuxからのクロスビルドを前提にしない。

## 製品情報とアイコン

- 製品名および発行元は`Hakamap`とする。
- 製品説明は「墓地マップ管理アプリ」とする。
- 製品バージョンは`メジャー.マイナー.パッチ`形式とし、例を`1.0.0`とする。
- 配布するインストーラーのファイル名は`Hakamap-<バージョン>.exe`とする。
- アプリケーションとインストーラーには同じ`.ico`ファイルを使用する。
- `.ico`ファイルには16ピクセルから256ピクセルまでの複数サイズを含める。
- 正式なアイコンが完成するまでは同じ要件を満たす仮アイコンを使用し、パッケージ設定から
  1か所を差し替えることで両方へ反映できる構成にする。
- Windowsのアップグレード用製品識別子は初回リリースから固定し、バージョン更新のたびに
  別アプリケーションとして重複登録されないようにする。

## ショートカットと起動

- スタートメニューへHakamapを登録する。
- デスクトップにHakamapのショートカットを作成する。
- ショートカットからランチャーを起動し、Spring Bootの準備完了後に既定ブラウザを開く。
- 2回目の起動では別バックエンドを開始せず、既存のHakamap画面を開く。

## 更新

- MVPでは自動アップデート機能を設けない。
- 新しいバージョンは、利用者が新版の`.exe`インストーラーを実行して上書き更新する。
- 更新および再インストールでは、プロジェクトフォルダ、バックアップ、復旧データ、
  カタログ、およびログを自動削除しない。
- 更新後に古いプロジェクト形式を開く場合は、プロジェクトデータのスキーマ変換手順に従う。

## コード署名

- MVPの`.exe`インストーラーにはコード署名証明書を使用せず、未署名で配布する。
- 未署名のためWindows SmartScreenなどの警告が表示される可能性を、ダウンロードおよび
  インストール手順で利用者へ事前に案内する。
- 自己署名証明書を正式配布へ使用しない。
- 将来、配布対象や運用方針が変わった場合にAuthenticode署名を追加できるよう、署名工程を
  パッケージ作成の後段へ追加可能な構成にする。
- 将来署名を導入する場合も、秘密鍵や証明書のパスワードをGitへ保存しない。

## アンインストールとユーザーデータ

- アプリケーションのインストール先と、プロジェクトおよび`%LOCALAPPDATA%\Hakamap`を分離する。
- アンインストールでは、プロジェクト、バックアップ、復旧データ、カタログ、およびログを
  自動削除しない。
- アプリ内の「この端末の管理データを削除」では、カタログ、復旧、一時、ログ、および
  実行時情報だけを削除し、プロジェクト、バックアップ、およびごみ箱は削除しない。
- プロジェクト本体は、アプリ内で各プロジェクトをごみ箱へ移動し、完全削除を明示した場合だけ削除する。

## 検証要件

- JavaがインストールされていないWindows 11環境へユーザー単位でインストールし、起動できることを確認する。
- スタートメニューとデスクトップのショートカットから起動できることを確認する。
- 新版インストーラーによる上書き更新後も、プロジェクトとカタログを利用できることを確認する。
- インストーラーのファイル名、製品情報、およびアイコンが定義どおりであることを確認する。
- 異なるバージョンで固定のアップグレード用製品識別子が使用され、別アプリとして
  重複登録されないことを確認する。
- アンインストール後もユーザーデータが残り、再インストール後に再利用できることを確認する。
- 端末管理データの削除後も、プロジェクト、バックアップ、およびごみ箱内データが残ることを確認する。
- 管理者権限を持たない最低環境のWindowsユーザーでインストール、起動、更新、および
  アンインストールを確認する。
- 未署名インストーラーで表示されるWindowsの警告と、利用者向け手順の内容を確認する。
