# データ・検索機能要件

## 墓所

- 墓所の内部主キーには、作成後に変更されないUUIDを使用する。
- 利用者が認識する墓所の業務キーは、「エリア」と「管理番号」の組み合わせとする。
- 同一プロジェクト内で「エリア」と「管理番号」の組み合わせを重複させない。
- 同じ管理番号でも、エリアが異なる場合は登録できる。
- エリアまたは管理番号を変更しても、人物、写真、添付ファイルなどとの関連を維持する。
- APIのリソース識別には内部UUIDを使用し、画面では原則として業務キーを表示する。
- 墓所名を登録できるようにする。
- 墓所名は任意入力とする。
- 墓所名を登録するときは前後の半角および全角空白を除去する。
- 地図上の墓所ラベルは、管理番号、墓所名、両方、または非表示から選択できるようにする。
- 組名は人物ごとではなく、墓所単位で1つ設定する。
- 組名は自由入力ではなく、プロジェクトの組名マスターから選択する。
- 墓所の更新日時は、墓所情報を変更したときにシステムが自動更新する。

## 人物

- 1つの墓所に複数人を関連付けられるようにする。
- 人物は墓所に紐づけて作成し、墓所に紐づかない人物を先に作成できないようにする。
- 人物の標準入力項目として、氏名と戒名を設ける。
- 人物の内部主キーには、作成後に変更されないUUIDを使用する。
- 氏名と戒名はそれぞれ任意入力とする。
- 人物を登録するときは、氏名と戒名の少なくとも一方を必須とする。
- 氏名と戒名を登録するときは、前後の半角および全角空白を除去する。
- 氏名および戒名の重複を許可する。
- 人物の更新日時は、人物情報を変更したときにシステムが自動更新する。

## 墓所の削除と復元

- 墓所を削除するときは、関連する人物および添付ファイルも削除されることを示した
  確認画面を表示する。
- 削除確認後はごみ箱へ移動せず即時削除する。
- Undo履歴が残っている間は、削除した墓所と関連データをUndoで復元できるようにする。
- アプリケーション終了後はUndo履歴を破棄し、削除を確定する。
- 削除後は、その墓所が使用していた「エリア＋管理番号」を再利用できるようにする。

## 人物の削除と復元

- 墓所を残したまま、関連する人物を1人ずつ削除できるようにする。
- 人物を削除するときは確認画面を表示する。
- 削除確認後はごみ箱へ移動せず即時削除する。
- Undo履歴が残っている間は、削除した人物をUndoで元の墓所へ復元できるようにする。
- アプリケーション終了後はUndo履歴を破棄し、削除を確定する。

## エリア

- エリアは墓地プロジェクトごとのマスターとして管理する。
- エリア名は必須とする。
- エリア名を登録するときは前後の半角および全角空白を除去する。
- エリア名は同一プロジェクト内で一意とする。
- エリアの新規作成時と名称変更時に重複を検査し、重複する場合は保存しない。
- 墓所の所属エリアは地図上の位置から自動判定する。
- 利用者が墓所の所属エリアを手動で上書きする機能はMVPに含めない。
- エリアは地図上の長方形の区画と関連付ける。
- 異なるエリアの長方形同士を重ねて登録できないようにする。
- エリア名の変更を許可し、関連する墓所の業務キー表示へ即時に反映する。
- 墓所側にエリア名の文字列を重複して保持しない。
- エリアを削除するときは確認画面を表示し、確認後はごみ箱へ移動せず即時削除する。
- エリアを削除しても、そのエリアに属していた墓所は削除せず未割当状態にする。
- Undo履歴が残っている間は、削除したエリアと墓所の所属をUndoで復元できるようにする。
- エリア外に配置されている墓所は利用者への警告対象とする。
- エリア範囲外への配置が必要な場合に対応するため、警告後も墓所を登録および保存できるようにする。

## 管理番号と業務キー

- 管理番号は必須とする。
- 管理番号は数字だけに限定せず、漢字、英字、数字、ハイフンなどを使用できるようにする。
- 管理番号の先頭のゼロは削除せず、入力された表記を保持する。
- 管理番号を登録するときは前後の半角および全角空白を除去する。
- 管理番号の重複判定では、全角と半角、および英字の大文字と小文字を区別しない。
- 業務キーの一意性は、変更可能なエリア名ではなく、内部のエリアUUIDと正規化した
  管理番号の組み合わせで判定する。
- エリア名を変更しても、墓所の一意性と関連データを維持する。

## 拡張項目

- 墓所および人物は、将来の項目追加に対応できる拡張項目を持つ。
- 拡張項目定義は墓地プロジェクトごとに管理する。
- 初期版では、1行文字列、複数行文字列、整数、小数、日付、真偽値、単一選択、
  および複数選択のデータ型を使用できるようにする。
- 日付時刻型は初期版に含めない。
- 拡張項目の作成時に、適用先として墓所または人物のどちらか一方を選択する。
- 1つの拡張項目定義を墓所と人物で共用せず、両方で必要な場合は適用先ごとに作成する。
- 同じ適用先では、利用停止中の項目を含めて同名の拡張項目を登録できないようにする。
- 適用先が異なる場合は、同じ項目名を登録できるようにする。
- 項目名の重複判定では、前後空白、全角と半角、および英字の大文字と小文字を正規化する。
- 拡張項目定義には、項目名、説明、データ型、必須・任意、表示順、および初期値を持たせる。
- 初期値は、選択したデータ型として有効な値だけを設定できるようにする。
- 選択肢型の項目は、定義されたマスターデータからプルダウンなどで選択できるようにする。
- 項目名、説明、表示順、および初期値は、項目作成後も変更できるようにする。
- データ型は、その項目の値が1件も登録されていない場合だけ変更できるようにする。
- 任意項目から必須項目への変更は、既存データの全件に有効な値が登録されている場合だけ許可する。
- 必須項目から任意項目への変更は許可する。
- 値が登録されている拡張項目は完全削除できず、利用停止にできるようにする。
- 利用停止した拡張項目は、新規登録および通常の編集フォームへ表示しない。
- 利用停止した拡張項目の既存値は保持し、詳細画面での確認と検索を可能にする。
- 値が1件も登録されていない拡張項目だけを完全削除できるようにする。
- 利用停止した拡張項目を再有効化できるようにする。

## マスターデータ管理

- 組名や選択肢型拡張項目の選択肢を、プロジェクトごとの管理画面で管理できるようにする。
- 選択肢を追加、名称変更、並べ替え、利用停止、および再有効化できるようにする。
- 利用停止した選択肢は新規入力では選択できないが、既存データとの関連は維持する。
- 選択肢の名称を変更すると、その選択肢を参照する全データの表示へ自動的に反映する。
- 選択肢を利用している各データに名称文字列を複製せず、マスターへの参照として保持する。
- マスター同士の統合や、既存データの一括再割り当ては初期版の画面機能に含めない。

## プロジェクト間の設定コピー

- 拡張項目定義と選択肢マスターを、別の墓地プロジェクトからコピーできるようにする。
- コピーには、項目名、データ型、表示順、必須・任意、選択肢、および選択肢の
  利用状態を含める。
- 墓所、人物、地図、画像などの実データはコピーしない。
- コピーした設定はコピー元から独立させ、コピー後に自動同期しない。
- コピー先の既存設定を自動的に上書きしない。
- 同名項目が存在する場合は、利用者が「スキップ」または「別名で追加」を選択できるようにする。

## 検索

- 墓所の備考を除き、利用者が値を登録できるすべての標準項目および拡張項目を検索対象にする。
- 墓所の備考はMVPの検索対象に含めない。
- 文字列項目は部分一致で検索する。
- 数値および日付項目は、FromとToの片方または両方を指定して範囲検索できるようにする。
- 選択肢項目は、マスターの選択肢を指定して絞り込めるようにする。
- 異なる項目に指定した検索条件はAND条件で組み合わせる。
- 同じ選択肢項目で複数の値を指定した場合は、その値をOR条件で組み合わせる。
- FromとToは、同じ数値または日付項目に対する1つの範囲条件として扱う。
- 条件を指定していない項目は、検索結果の絞り込みに使用しない。
- 利用者がすべての検索条件を一括でクリアできるようにする。
- 利用者が任意のAND・OR条件を組み立てる高度な検索画面は、初期版に含めない。
- 文字列を登録するときは前後の半角および全角空白を除去し、文字列内の空白は保持する。
- 文字列検索では英字の大文字と小文字を区別しない。
- 文字列検索では全角と半角を区別しない。
- 文字列検索では文字列内を含む半角および全角空白を無視する。
- 文字列検索ではひらがなとカタカナを区別しない。
- 検索用の正規化によって保存値を変更せず、画面には利用者が登録した表記を表示する。
- 漢字の異体字および旧字体の自動変換は、初期版に含めない。
- 検索結果には、一致した人物と、その人物が属する墓所の関係が分かる情報を表示する。
- 検索結果一覧は、利用者が閉じるか検索条件をクリアするまで表示したままにする。
- 検索後は最初の検索結果へ地図をズームする。
- 検索結果から墓所を選択すると、該当する墓所を地図上で選択し、確認しやすい位置へ表示する。
