# プロジェクト管理機能要件

## プロジェクトの作成と変更

- プロジェクト名は必須とする。
- 背景画像を登録しなくてもプロジェクトを作成できるようにする。
- 背景画像はプロジェクト作成後に追加および変更できるようにする。
- プロジェクト作成時にエリアを自動作成しない。
- エリアはプロジェクト作成後、利用者が必要なタイミングで追加する。
- プロジェクト名は作成後も変更できるようにする。
- プロジェクトの内部識別子には、作成後に変更されないUUIDを使用する。
- プロジェクト名を変更しても内部識別子を変更しない。
- プロジェクト名の変更はUndoおよびRedoの対象とする。
- プロジェクトを作成または更新した利用者の情報は保持しない。
- プロジェクトには作成日時と更新日時を保持する。
- 利用者がプロジェクトごとに保存場所を指定できるようにする。

## 起動時のプロジェクト表示

- 利用者がデフォルトプロジェクトを明示的に設定できるようにする。
- デフォルトプロジェクトが有効な場合は、アプリケーション起動時にそのプロジェクトを
  直接開く。
- 次の場合はプロジェクト選択画面を表示する。
  - デフォルトプロジェクトが設定されていない場合
  - デフォルトプロジェクトが削除されている場合
  - デフォルトプロジェクトを読み込めない場合
- プロジェクト選択画面には、プロジェクト名、作成日時、および更新日時を表示する。

## 複数プロジェクト

- 同時に開けるプロジェクトは1件だけとする。
- 別のプロジェクトを開く際に未保存変更がある場合は確認ダイアログを表示する。
- 確認ダイアログでは、保存、破棄、およびキャンセルを選択できるようにする。

## プロジェクトの削除と復元

- プロジェクトの削除前に確認画面を表示する。
- 削除したプロジェクトは即時に完全削除せず、ごみ箱へ移動する。
- 墓所、人物、添付ファイル、エリア、背景、およびプロジェクト設定をプロジェクト一式
  としてごみ箱へ移動する。
- ごみ箱へ移動したプロジェクト一式を復元できるようにする。
- プロジェクトの完全削除は、ごみ箱内で利用者が明示的に実行する。
- プロジェクトを完全削除すると、そのプロジェクトに属するデータ一式を削除する。
- デフォルトプロジェクトをごみ箱へ移動した場合は、デフォルト設定を解除する。
- デフォルト設定の解除後、別のプロジェクトを自動的にデフォルトへ設定しない。
- 削除したプロジェクトを復元しても、デフォルト設定を自動的に復元しない。
- 復元したプロジェクトをデフォルトにする場合は、利用者が明示的に設定する。
