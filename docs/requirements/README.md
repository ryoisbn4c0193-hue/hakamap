# Hakamap要件文書

利用者から見た機能と振る舞いは、このディレクトリの文書を正とする。技術構成、保存方式、
セキュリティ境界、および非機能要件の実現方針は`../design`配下で管理する。

## 更新時の扱い

- 確定した利用者要件を該当する機能要件へ記載する。
- 性能、動作環境、ログ、復旧、保存方式などは`../design`配下へ記載する。
- 既存の確定事項と新しい検討内容が矛盾する場合は、どちらかを推測で採用せず、
  利用者へ確認する。
- 「削除」と「ごみ箱へ移動」、「バックアップ」と「自動復旧」、「想定最大件数」と
  「入力上限」を区別する。
