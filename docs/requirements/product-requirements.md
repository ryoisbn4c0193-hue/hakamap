# 製品機能要件

## 目的

Hakamapの初期版で対象とする利用者、解決する課題、および主要な利用フローを定める。
技術方式に関する決定は`docs/design`配下の設計文書で扱う。

## 対象利用者と課題

- 最優先の利用者は、寺院の墓地管理者とする。
- 紙の墓地図と埋葬記録をローカル環境でデジタル化できるようにする。
- 氏名などの登録情報から墓所を検索し、目的の墓所を地図上ですぐに確認できるようにする。
- 単一ユーザー向けとし、利用時にインターネット接続や外部サービスを必要としない。

## 初期登録の流れ

1. 利用者が墓地の全体図を画像またはPDFから背景として取り込む。
2. 利用者が背景上へ墓所などの地図要素を配置する。
3. 利用者が各墓所と、その墓所に属する人物の情報を画面から入力する。
4. 利用者が登録情報を検索し、検索結果から該当する墓所を地図上で確認する。

## 初期版の範囲

- 人物情報は画面から一件ずつ入力できるようにする。
- ExcelやCSVなどからの人物情報の一括取り込みは、初期版の必須機能に含めない。
- 1つの墓所に複数人を登録できるようにする。
- 複数の墓地は、独立した墓地プロジェクトとして管理できるようにする。
- 写真およびPDFを、墓所管理に関係するローカルファイルとして登録できるようにする。
- 印刷、PDF出力、および画像出力をオフラインで行えるようにする。
- プロジェクトのエクスポートおよびインポートを行えるようにする。

## MVP対象外・将来拡張

- 墓所およびエリアの多角形
- 墓所を回転するUI
- ログインおよびユーザー管理
- 複数利用者による同時編集
- アプリケーション独自のデータ暗号化
- Windows 11以外のOS対応
- 墓所の備考検索
- 墓所の所属エリアを手動で上書きする機能
- グリッドスナップ

## 未決事項

- 対応するブラウザのバージョン
- 想定する人物数
- 地図背景の最大容量
- 人物情報の一括入出力を初期版へ含めるか
- プロジェクトのエクスポート形式、圧縮方式、および拡張子
