# 地図編集機能要件

## 背景

- 墓地全体の図面を、利用者が所有する画像またはPDFから取り込めるようにする。
- 背景取り込みの詳細は`docs/design/map-background-import.md`に従う。

## 墓所

- 墓所は一点を示すピンではなく、長方形として地図上に配置する。
- 墓所の位置、大きさ、および回転角度を地図座標として保存する。
- 背景画像を移動または再調整しても、墓所の保存座標を自動的に変更しない。
- 墓所上に「エリア＋管理番号」と墓所名をラベルとして表示する。
- 地図上で墓所を削除するときは確認画面を表示し、墓所と関連データをごみ箱へ移動する。
- ごみ箱へ移動した墓所は通常の地図上に表示しない。
- 墓所を復元した場合は、保存されていた位置、大きさ、および回転角度で地図上へ戻す。

## エリア

- エリアを地図上の多角形の区画として描画できるようにする。
- 地図上のエリアは、プロジェクトごとのエリアマスターと関連付ける。
- 1つのエリアにつき、地図上へ登録できる多角形は1つだけとする。
- 同じエリアに属する範囲が離れている場合は、それぞれを別エリアとして登録する。
- 異なるエリアの多角形同士が重なる配置は禁止し、保存時に検証エラーを表示する。
- エリアの多角形は作成後も形状を変更できるようにする。
- 形状変更によって関連する墓所がエリア範囲外になった場合は、利用者へ警告する。
- エリアを利用停止にしても、既存墓所との関連と地図上の表示を維持する。
- 墓所のエリアは利用者が明示的に選択する。
- 墓所が選択したエリアの多角形の範囲外にある場合は、利用者へ警告する。
- 範囲外であることを理由に保存を禁止せず、利用者は警告を確認したうえで登録できる。

## 地図要素

- 初期状態では、墓所、通路、入口、および建物（本堂）を配置できるようにする。
- 初期状態で使用する形状は次のとおりとする。
  - 墓所: 長方形
  - 通路: 折れ線に幅を持たせた形状
  - 入口: 地図上の一点を示すアイコン
  - 建物（本堂）: 長方形または多角形
- 地図要素の種類はプロジェクトごとのマスターとして管理する。
- 利用者が地図要素の種類を追加できるようにする。
- 種類を追加するときは、長方形、多角形、線、またはアイコンから形状を選択できるようにする。
- 種類の名称変更、並べ替え、利用停止、および再有効化に対応する。
- 種類の名称変更は、その種類を参照するすべての地図要素の表示へ自動的に反映する。

## 未決事項

- 地図要素ごとの色、線、透明度、アイコン、およびラベルの設定
- 選択、移動、サイズ変更、回転、複製、削除の詳細操作
- スナップ、整列、ズーム、およびパンの操作
- UndoとRedoの対象範囲および履歴の保持期間
