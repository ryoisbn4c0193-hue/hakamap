# 地図編集画面・操作要件

## 画面構成

- 地図編集画面は、左サイドバー、中央地図キャンバス、および右プロパティパネルの
  3領域を基本とする。
- 低い画面解像度でも地図の操作領域を確保できるよう、左右のパネルを折りたためるようにする。

## 左サイドバー

- エリア分類として「すべて」、各エリア、および「未割当」を表示する。
- 管理状態として「未割当」、「未採番」、および「情報未完成」を表示する。
- 未採番には、管理番号が設定されていない墓所を表示する。
- 情報未完成には、所属エリアまたは管理番号など、墓所の完成条件を満たしていない墓所を表示する。

## 右プロパティパネル

- 墓所を選択すると右プロパティパネルを表示する。
- 地図を表示したまま墓所情報を編集でき、通常の編集では別画面へ遷移しない。
- 右プロパティパネルはタブ形式とし、基本情報、人物、添付、および履歴を表示する。
- タブ名称と詳細な配置は画面設計で調整できるものとする。
- 墓所の人物タブには人物一覧を表示し、人物の追加、編集、および削除を行えるようにする。
- 人物の追加および編集は専用ダイアログで行う。
- 墓所の添付タブには添付ファイル一覧を表示し、追加、削除、およびプレビューを
  行えるようにする。
- 地図上の空白をクリックすると選択を解除し、右プロパティパネルを閉じる。
- 墓所のダブルクリックには機能を割り当てない。
- 墓所を選択せずに人物操作を開始した場合は「墓所を選択してください。」と案内する。
- 墓所を選択せずに添付操作を開始した場合は「墓所を選択してください。」と案内する。

## 複数選択時の表示

- 2件以上の地図要素を選択した場合は、右プロパティパネルを複数選択専用表示へ切り替える。
- 複数選択専用表示には「○件選択中」と表示し、個別墓所の情報を表示しない。
- 複数選択時に利用可能な操作を表示する。
- MVPで墓所の複数選択時に利用できる一括操作は、移動、コピー、および削除とする。

## 選択操作

- クリックで単一の地図要素を選択する。
- Ctrlキーを押しながらクリックすると、選択対象を追加または解除する。
- 地図上のドラッグ操作で矩形選択する。
- Ctrlキーを押しながらドラッグすると、現在の選択を維持したまま矩形選択を追加する。
- Shiftキーは選択の追加または範囲選択には使用しない。

## キーボード操作

- テキスト入力中でない場合、`+`キーで拡大、`-`キーで縮小、`0`キーで全体表示を行う。
- Spaceキーを押しながら左ドラッグすると地図をパンする。
- 選択中の移動可能な地図要素は、矢印キーで地図座標1単位ずつ移動できるようにする。
- Shiftキーと矢印キーを同時に押すと、地図座標10単位ずつ移動できるようにする。
- キーボードによる移動量はズーム率および画面ピクセル数に依存しない。
- 複数選択中も同じ単位で一括移動できるようにする。
- 地図要素をドラッグ移動中にShiftキーを押すと、ドラッグ開始位置を基準に水平または
  垂直のうち移動量が大きい方向へ移動を制限する。
- 地図要素のサイズ変更中にShiftキーを押すと、ドラッグ開始時の縦横比を維持する。
- 背景は通常時にX・Yを独立して拡縮し、Shiftキーを押した場合だけ縦横比を維持する。
- ドラッグ中にShiftキーを押す、または離した場合も制約のプレビューへ即時に反映する。
- テキスト入力欄にフォーカスがある場合は、矢印キーによる地図要素の移動を行わない。
- テキスト入力欄にフォーカスがある場合は、ズームおよびパンのキーボードショートカットも行わない。

## UndoとRedo

- UndoおよびRedoは、データを変更する操作だけを対象とする。
- 作成、移動、サイズ変更、削除、復元、コピー、エリア編集、背景変形、および
  プロジェクト名変更を対象に含める。
- ズーム、パン、検索、選択状態、ラベル表示切り替え、およびサイドバー表示状態は
  対象に含めない。
- 保存を実行してもUndoおよびRedoの履歴を削除しない。
- 履歴は開いているプロジェクトの編集セッション中だけ保持し、プロジェクトを閉じる、
  別プロジェクトへ切り替える、またはアプリケーションを終了するときに破棄する。
- 再起動後に履歴を復元しない。
- 履歴は最大100操作とし、上限を超えた場合は古い履歴から削除する。
- Undo後に新しい編集を行った場合はRedo履歴を破棄する。
- 複数対象への一括操作は、操作全体を1操作として履歴へ記録する。
- 検証エラーなどで失敗した操作は履歴へ記録しない。
- 履歴タブには最大100件の操作を新しい順に表示する。
- 履歴には操作日時、個人情報を含まない操作説明、Undo済みまたは適用中の状態、および
  手動保存時点との前後関係を表示する。
- 保存操作自体はUndo履歴へ追加せず、「ここまで保存済み」の位置を表示する。
- 履歴の任意時点へ直接移動する機能と、履歴を個別削除または全消去する機能はMVPに含めない。

## 保存と未保存変更

- プロジェクトの保存は手動とする。
- 未保存変更がある状態で画面移動、プロジェクト切り替え、またはアプリケーション終了を
  行おうとした場合は確認ダイアログを表示する。
- 確認ダイアログでは、操作内容に応じて保存、破棄、およびキャンセルを選択できるようにする。
- 保存してもUndoおよびRedo履歴を維持する。
- 墓所、エリア、人物、および添付ファイルの即時削除は、編集中の状態から直ちに削除する
  ことを意味し、削除操作を未保存変更として扱う。
- 削除後に保存すると、削除をプロジェクトデータへ確定する。
- 削除後、保存前に変更を破棄した場合は、最後に保存した状態へ戻して削除を取り消す。
- 保存前の削除はUndoでも取り消せるようにする。
- 削除後に保存した場合も、現在のプロジェクトを開いている間はUndoで削除を取り消せるようにする。
- 保存済みの削除をUndoした場合、その復元操作を新たな未保存変更として扱う。
- アプリケーション終了時にUndoおよびRedo履歴を破棄し、次回起動時は最後に保存した状態を表示する。

## アプリケーションの終了と再表示

- アプリケーション画面のメニューに、Hakamap全体を明示的に終了する操作を設ける。
- ブラウザのタブまたはウィンドウを閉じただけではHakamapを終了しない。
- 初回起動時に、タブを閉じても終了しないこと、終了には画面内の終了操作を使用すること、
  および画面を閉じた後はショートカットから開き直せることを案内する。
- ブラウザ画面を閉じた後は、デスクトップまたはスタートメニューのHakamapショートカットを
  再実行して画面を開き直せるようにする。
- 終了方法と画面の開き直し方をヘルプから確認できるようにする。
- MVPではタスクトレイへ常駐しない。
