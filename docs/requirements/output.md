# 印刷・出力機能要件

## 対応する出力

- 印刷、PDF出力、および画像出力をオフラインで実行できるようにする。
- 出力範囲は、現在表示中の範囲または選択したエリアから選べるようにする。
- 地図上で現在選択されているラベル表示設定を出力へ反映する。
- 印刷およびPDF出力の用紙サイズはA4とA3に対応する。
- 用紙の向きは縦と横から選択でき、初期値をA4・横とする。
- 余白は上下左右10mm固定とする。
- 選択した出力範囲の縦横比を維持し、墓所とラベルを含む全体が印刷可能領域内へ収まるよう
  自動的に縮尺を調整する。
- 実寸縮尺の指定はMVPに含めない。
- ヘッダー、フッター、およびページ番号はMVPでは出力しない。
- 印刷とPDF出力で同じレイアウト計算を使用する。
- PDFは選択した用紙サイズ、向き、および余白を反映した1ページとして出力し、ページ分割しない。
- 画像はPNG形式で、選択した用紙サイズ、向き、および余白を反映した1枚として出力する。
- 画像の解像度は300 DPI相当で固定し、背景色は白とする。
- 印刷、PDF出力、または画像出力の実行前に共通のプレビューを表示する。

## プロジェクトのエクスポートとインポート

- 墓地プロジェクトを`.hakamap`拡張子の単一ファイルとしてエクスポートおよびインポートできるようにする。
- `.hakamap`ファイルはZIP形式とし、プロジェクトの構造化データ、背景、および添付を含める。
- プロジェクト内のバックアップ、ロックファイル、復旧データ、一時ファイル、キャッシュ、
  およびログはエクスポートへ含めない。
- エクスポートファイルを暗号化またはパスワード保護しない。
- 未保存変更がある場合は、「変更を保存してエクスポート」または「キャンセル」を選択できるようにする。
- 最後に保存した古い状態だけをエクスポートする機能はMVPに含めない。
- 保存に失敗した場合はエクスポートを開始しない。
- 保存先が切断されている場合は、再接続または別プロジェクトとしての保存が成功するまで
  エクスポートを開始しない。
- エクスポート成功後も現在のプロジェクトを開いた状態にする。
- エクスポート操作自体はプロジェクトの更新日時およびUndo／Redo履歴を変更しない。
- プロジェクトを開いている状態でインポートする場合、未保存変更について保存、破棄、または
  キャンセルを選択できるようにする。
- インポート前に現在のプロジェクトを閉じ、成功後は取り込んだプロジェクトを開く。
- 通常のインポートでは、プロジェクトUUID、名称、作成日時、更新日時、および配下データのUUIDを維持する。
- 同じUUIDの利用可能なプロジェクトが登録済みの場合は、重複登録の拒否または
  「別プロジェクトとして取り込む」を選択できるようにする。
- インポート失敗時は不完全なプロジェクトを残さず、プロジェクト一覧画面へ戻る。
- インポートおよび別プロジェクトとしての取り込みで、デフォルトプロジェクト設定を変更しない。
- `.hakamap`によるプロジェクト全体のエクスポートおよびインポートには人物データを含める。
- 人物情報だけをCSVまたはExcelで一括入出力する機能はMVPに含めない。
