# 写真・添付ファイル機能要件

## 対応形式

- 初期版では、画像およびPDFを写真・添付ファイルとして登録できるようにする。
- 初期版のホワイトリストには、PNG、JPEG、WebP、およびPDFを含める。
- 添付PDFは1ページだけのPDFに対応する。
- 複数ページPDFは業務エラーとして拒否する。
- 添付PDFは画像へ変換し、画像と同様に扱う。
- SVG、GIF、TIFF、およびBMPは、初期版のホワイトリストに含めない。
- 登録可能なファイル形式は、許可する形式だけを列挙したホワイトリストで管理する。
- 拡張子だけでなくファイル内容を検査し、ホワイトリストにない形式や、拡張子と内容が
  一致しないファイルを拒否する。
- ホワイトリストはソースコードまたはアプリケーションに同梱する設定として管理する。
- 利用者が画面や外部設定ファイルからホワイトリストを編集する機能は設けない。
- 対応形式を変更する場合は、ソースコードまたは同梱設定を変更してアプリケーションを更新する。
- PDF内のスクリプト、フォーム、埋め込み添付ファイル、動画、および音声を実行しない。

## 保存

- 墓所へ写真および添付ファイルを登録できるようにする。
- 墓所に紐づかない添付ファイルを先に登録できないようにする。
- 登録したファイルは、関連する構造化データとともにバックアップおよび復元の対象にする。
- 添付ファイルはプロジェクトフォルダ内のアセットファイルとして保持する。

## 容量制限

- 添付ファイルは1ファイル最大25MBとする。
- 添付画像は最大4,000万画素とし、幅および高さをそれぞれ最大15,000pxとする。
- プロジェクト全体の容量には固定上限を設けない。
- 上限を超えるファイルは自動縮小せず、理由を表示して追加を拒否する。
- 追加時は拡張子だけでなく、実ファイル内容、MIMEタイプ、および画像の画素数を検証する。
- 追加および保存前に必要容量を見積もり、保存先の空き容量が不足する場合は処理を開始しない。
- ファイル登録前に必要な空き容量を確認し、保存を安全に完了できない場合は登録を開始しない。
- 1つの墓所へ登録できる添付ファイルは20件までとする。
- 20件を超える追加は業務エラーとし、ファイルを登録しない。
- 多数のファイルが登録されている場合は、一覧を一定件数ずつ読み込んで表示する。

## 表示情報

- 各ファイルに表示名、説明、登録日時、更新日時、および表示順を持たせる。
- 表示名は利用者が変更できるようにし、登録時の初期値には元のファイル名を使用する。
- 表示名は最大100文字とし、改行を許可しない。
- 元のファイル名はディレクトリなどのパスを除いたファイル名部分だけを保持する。
- 元のファイル名は最大255文字とする。
- 説明は任意入力とする。
- 説明は最大500文字とし、改行を許可する。
- 登録日時はファイル登録時にシステムが自動設定する。
- 更新日時は表示名または説明を変更したときにシステムが自動更新する。
- ファイル形式とファイル容量はシステムが取得し、読み取り専用の情報として表示する。
- 利用者が複数ファイルの表示順を並べ替えられるようにする。
- 新規添付は墓所内の既存添付の末尾へ追加する。
- ドラッグ＆ドロップに加え、キーボードで操作できる「上へ」および「下へ」を提供する。
- 添付の並べ替えはUndoおよびRedoの対象とする。
- 複数ファイルを一度に追加した場合は、ファイル選択ダイアログから受け取った順で末尾へ追加する。
- 添付削除後は残った表示順を詰め、Undoで削除を取り消した場合は元の位置へ戻す。
- 表示名または説明を変更しても表示順を変更しない。
- 表示順をインポート、エクスポート、バックアップ、および復元で維持する。
- 表示名、元のファイル名、および説明の文字数はUnicodeコードポイント単位で数える。
- 制御文字は拒否する。ただし、説明の改行文字は許可する。
- 上限を超える値は自動的に切り捨てず、入力エラーとして保存しない。

## プレビュー

- 画像のサムネイルを生成し、選択するとアプリ内のダイアログで拡大表示する。
- サムネイルの表示枠は横240px、縦180pxとする。
- 元画像の縦横比を維持して表示枠内に収め、小さい画像は拡大しない。
- サムネイルはWebP形式、品質80で生成する。
- サムネイルを生成できない場合は、ファイル形式に応じた汎用の画像またはPDFアイコンを表示する。
- 設定画面からサムネイルキャッシュを削除でき、必要になった場合は自動的に再生成する。
- PDFは1ページを画像へ変換し、画像と同じプレビュー画面で表示する。
- PDFのページ移動およびPDF内リンクの操作機能は設けない。

## 削除と復元

- 墓所を残したまま、登録したファイルを1件ずつ削除できるようにする。
- ファイルを削除するときは確認画面を表示する。
- 削除確認後はごみ箱へ移動せず即時削除する。
- Undo履歴が残っている間は、削除したファイルをUndoで元の墓所へ復元できるようにする。
- プロジェクトを閉じる、切り替える、またはアプリケーションを終了するときにUndo履歴を
  破棄し、以後は保存済みの削除をUndoできない。
