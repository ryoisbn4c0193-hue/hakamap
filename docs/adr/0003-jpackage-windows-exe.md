# ADR 0003: jpackageによるユーザー単位のWindows EXEを採用する

## 状態

採用

## コンテキスト

HakamapはWindows 11向けのオフラインアプリケーションであり、利用者へJava、Node.js、
または開発ツールの導入を要求せず配布する必要がある。プロジェクトとアプリ管理データは
インストール先の外に保存し、アプリの更新や削除から保護する必要がある。

## 決定

- `jlink`で必要なモジュールだけを含むJava 21ランタイムを作成する。
- `jpackage`でランタイムを同梱したWindows `.exe`インストーラーを作成する。
- ユーザー単位でインストールし、原則として管理者権限を要求しない。
- スタートメニューとデスクトップへショートカットを作成する。
- MVPでは自動アップデートを設けず、新版インストーラーによる上書き更新とする。
- 更新、再インストール、およびアンインストールではユーザーデータを自動削除しない。
- MVPではコード署名証明書を使用せず、未署名の`.exe`として配布する。SmartScreen警告の
  可能性を案内し、将来署名工程を追加可能な構成にする。

## 理由

- Java 21ランタイムをアプリへ同梱でき、利用者のJava環境へ依存しない。
- 既存のJavaおよびSpring Boot構成と同じビルド境界でWindows向け成果物を作成できる。
- ユーザー単位インストールにより、個人利用で管理者権限を要求しない運用にできる。
- アプリ本体とユーザーデータを分離し、更新とアンインストールによるデータ消失を防げる。

## 影響

- Windows向けインストーラーはWindows上でビルドおよび検証する必要がある。
- 同梱するJavaモジュールを、Spring Boot、Swingファイル選択、および画像・PDF処理を含めて特定する必要がある。
- 自動更新がないため、利用者が新版インストーラーを入手して実行する必要がある。
- 未署名のため、Windows SmartScreenなどの警告が表示される可能性がある。
- ユーザーデータを完全に削除する場合は、アンインストールとは別の明示操作が必要になる。
