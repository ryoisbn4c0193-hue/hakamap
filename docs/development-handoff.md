# 開発引き継ぎメモ

最終更新日: 2026-07-22

## 現在の目的

Hakamap MVPの要件定義を確定し、GitHub上の文書を唯一の正しい情報源として整備している。
2026-07-22に外部のChatGPTで検討した内容と、その後の文書レビュー結果が連携され、
利用者の指示に従って最新要件へ反映した。

## 今回完了した作業

- プロジェクトの作成、デフォルト設定、起動時表示、切り替え、削除、および復元を定めた。
- 背景なしでのプロジェクト作成と、作成後の背景追加・変更を定めた。
- 複数ページPDFを背景として指定した場合は業務エラーとする仕様へ変更した。
- 墓所の一括作成、連続生成、複数選択、一括操作、重なり、コピー、および採番を定めた。
- MVPのエリアを長方形とし、墓所の所属を位置から自動判定する仕様へ変更した。
- 墓所、エリア、人物、添付ファイルはごみ箱を使わず即時削除し、Undo履歴中だけ
  復元できる仕様へ変更した。プロジェクト削除だけはごみ箱を使用する。
- 添付ファイルは墓所だけに紐づけ、1墓所20件までとする仕様へ変更した。
- 地図編集画面の3領域、プロパティパネル、選択操作、キーボード操作を定めた。
- 手動保存、未保存変更の確認、および最大100操作のUndo・Redoを定めた。
- 印刷、PDF出力、画像出力、およびプロジェクトのインポート・エクスポートを定めた。
- Windows 11の最低・推奨環境、5,000墓所を前提とする性能目標を定めた。
- 自動復旧、ログ、エラー処理、およびセキュリティ要件を定めた。
- プロジェクトの保存先を利用者が自由に指定する仕様へ変更した。
- 自動バックアップをプロジェクトの`backup`フォルダへ1日1回、3世代保存する仕様へ変更した。
- 最終採用技術、プロジェクト物理形式、インストーラー形式、および自動更新を未確定へ変更した。
- 入力項目とファイル容量の上限を未確定へ変更した。
- 添付PDFを1ページ限定とし、複数ページを拒否して画像としてプレビューする仕様へ変更した。
- 管理番号を墓所の完成条件とし、未採番および情報未完成の墓所を保存可能とした。
- 通路、入口、建物、および自由地図要素をMVP対象外へ変更した。
- 拡張項目、マスターデータ、およびプロジェクト間設定コピーを将来候補へ移動した。
- 詳細検索仕様と人物入力項目を未決または将来候補へ移動した。
- 背景とバックアップに関する未確認の設計判断を設計候補へ移動した。
- 即時削除、手動保存、変更破棄、およびUndoの関係を確定した。

## 次に行うこと

今回残った未決事項を、利用者への質問によって順次確定する。

- プロジェクト名の最大文字数と重複可否
- 各入力項目の最大文字数
- プロジェクトを単一ファイルにするかフォルダ構成にするか
- プロジェクトの内部データ形式、拡張子、添付ファイルの保持方式
- 地図背景および添付ファイルの容量・画素数上限
- 人物の入力項目とMVPの検索対象・検索方法
- 自動復旧の一時保存間隔、保存形式、および削除タイミング
- ログのファイルサイズ、世代数、および最大容量
- ネットワークドライブの排他制御と切断時の動作
- 印刷・PDF・画像出力の詳細
- 最終採用技術、アプリケーション構成、インストーラー形式、および自動アップデート
